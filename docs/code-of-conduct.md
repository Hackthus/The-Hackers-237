# Code de Conduite

Ce document définit les règles et valeurs qui régissent la communauté **The H@ckers 237**. En contribuant ou en utilisant ce projet, vous acceptez de respecter ces principes.


## Notre Engagement

The H@ckers 237 est un projet open source dédié à l'apprentissage de la cybersécurité offensive dans un cadre **éthique, légal et respectueux**. Nous nous engageons à maintenir un environnement sain, inclusif et bienveillant pour tous les participants, quels que soient leur niveau, leur origine ou leur expérience.


## Valeurs du Projet

- **Partage** - La connaissance se construit collectivement
- **Ethique** -  Toute technique documentée est destinée à des usages légaux et autorisés
- **Rigueur** -  Les contenus publiés sont vérifiés et sourcés
- **Respect** -  Chaque contributeur mérite considération et bienveillance
- **Transparence** -  Les décisions du projet sont ouvertes et documentées


## Comportements Attendus

- Utiliser un langage respectueux et professionnel
- Accepter les critiques constructives avec ouverture
- Citer les sources et respecter la propriété intellectuelle
- Tester les commandes avant de les documenter
- Signaler les erreurs ou inexactitudes plutôt que de les ignorer
- Aider les nouveaux contributeurs à s'intégrer
- Documenter uniquement dans un cadre **éthique et légal**


## Comportements Inacceptables

- Harcèlement, insultes ou attaques personnelles
- Publication de contenu discriminatoire ou offensant
- Partage de données personnelles sans consentement
- Documentation de techniques ciblant des infrastructures réelles sans autorisation
- Publication d'exploits 0-day non divulgués
- Inclusion de credentials réels ou de données sensibles dans les contributions
- Toute forme de plagiat sans attribution


## Utilisations Autorisées

Ce projet est conçu pour :

- L'apprentissage de la cybersécurité offensive
- La préparation aux certifications (OSCP, CPTS, PT1 etc..)
- Les tests d'intrusion dans des environnements **autorisés**
- La recherche en sécurité offensive
- Les labs et environnements de test dédiés (GOAD, HackTheBox, TryHackMe, etc..)


## Utilisations Interdites

Ce projet **ne doit pas** être utilisé pour :

- Attaquer des systèmes sans autorisation explicite
- Compromettre des infrastructures réelles
- Voler des données personnelles ou confidentielles
- Toute activité illégale au regard des lois en vigueur

---

::: danger Avertissement légal

L'utilisation de ces informations sur des systèmes sans autorisation explicite est **illégale** dans la plupart des pays et peut entraîner des poursuites pénales. Les auteurs du projet déclinent toute responsabilité en cas d'utilisation malveillante ou illégale du contenu documenté.

:::


## Signalement

Si vous êtes témoin d'un comportement contraire à ce code de conduite ou d'un contenu inapproprié :

1. Ouvrir une **issue** sur GitHub avec le label `code-of-conduct`
2. Contacter directement les mainteneurs via GitHub
3. Soumettre un rapport via les **Discussions GitHub**

Tout signalement sera traité avec confidentialité et sérieux.


## Conséquences

Les violations de ce code de conduite peuvent entraîner :

| Gravité | Conséquence |
|---------|-------------|
| Mineure | Avertissement écrit |
| Modérée | Retrait temporaire des droits de contribution |
| Grave   | Exclusion permanente du projet |
| Critique | Exclusion définitive si nécessaire |


## Responsabilités des Mainteneurs

Les mainteneurs du projet sont responsables de :

- Clarifier et appliquer ce code de conduite
- Prendre des mesures correctives appropriées en cas de violation
- Maintenir la confidentialité des signalements
- Faire évoluer ce document en fonction des besoins de la communauté


## Portée

Ce code de conduite s'applique à tous les espaces du projet :

- Dépôt GitHub (issues, pull requests, discussions)
- Documentation et contenu du wiki
- Toute communication liée au projet
